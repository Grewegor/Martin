# Сайт "Мартин" - Документация

## Общее описание

Полнофункциональный веб-сайт поклонников певца Мартина с системой авторизации пользователей, управлением данными и использованием AJAX для асинхронной загрузки и сохранения информации.

## Требования (выполнены)

✅ **Авторизация пользователей** - требуется вход для доступа к контенту  
✅ **Регистрация** - возможность создания новых аккаунтов  
✅ **Хранение данных** - пользователи и альбомы хранятся в JSON файлах  
✅ **Bootstrap-table** - таблица альбомов с интерактивными возможностями  
✅ **AJAX** - все данные загружаются/сохраняются без перезагрузки страницы  
✅ **Валидация на клиенте** - без использования внешних библиотек

## Функциональность

### 1. Система авторизации

#### Вход в систему (login.html)
- Форма входа с валидацией имени пользователя и пароля
- Демо-аккаунт:
  - **Имя пользователя**: `demo`
  - **Пароль**: `demo123`

#### Регистрация новых пользователей
- Форма регистрации с полями:
  - Имя пользователя (3-20 символов, буквы, цифры, подчеркивание)
  - Email (валидный формат)
  - Пароль (минимум 6 символов)
  - Подтверждение пароля
- Валидация на клиенте (без внешних библиотек)
- Проверка уникальности имени пользователя и email
- Сохранение в `data/users.json`

#### Управление сессией
- Токены сохраняются в `localStorage`
- Каждый токен имеет срок действия (7 дней)
- Проверка токена при загрузке защищенных страниц
- Кнопка выхода в правом верхнем углу

### 2. Защищенный контент

- **Главная страница (index.html)** - доступна только авторизованным пользователям
- **Страница тура (tour.html)** - доступна только авторизованным пользователям
- При попытке доступа без авторизации - редирект на страницу входа

### 3. Управление альбомами

#### Отображение альбомов
- Таблица (bootstrap-table стиль) с колонками:
  - Год выпуска
  - Название альбома
  - Действия (кнопка удаления)
- Данные загружаются из `data/albums.json` в формате:
```json
[
    {
        "year": "2026",
        "title": "чао"
    },
    {
        "year": "2025",
        "title": "пока лето не кончилось"
    }
]
```

#### Добавление альбомов
- Форма с полями для года и названия альбома
- Валидация на клиенте:
  - Год: 4 цифры (1900-2100)
  - Название: 2-100 символов
- Сохранение через AJAX (метод GET)
- Автоматическое обновление таблицы после добавления

#### Удаления альбомов
- Кнопка "Удалить" рядом с каждым альбомом
- Подтверждение перед удалением
- Сохранение через AJAX (метод POST)

## Валидация данных (клиент)

### Валидация при входе
- Имя пользователя: 3-20 символов
- Пароль: минимум 6 символов

### Валидация при регистрации
- Имя пользователя: 3-20 символов, только буквы/цифры/подчеркивание
- Email: корректный формат (regex)
- Пароль: минимум 6 символов
- Подтверждение: совпадение с паролем

### Валидация альбомов
- Год: 4 цифры, от 1900 до 2100
- Название: 2-100 символов

Все валидации реализованы на JavaScript без использования внешних библиотек.

## API endpoints

### POST `/api/login.php`
**Вход в систему**

Запрос:
```json
{
    "username": "demo",
    "password": "demo123"
}
```

Ответ (успех):
```json
{
    "success": true,
    "token": "abc123...",
    "username": "demo",
    "message": "Вы успешно вошли"
}
```

### POST `/api/register.php`
**Регистрация нового пользователя**

Запрос:
```json
{
    "username": "newuser",
    "email": "user@example.com",
    "password": "password123"
}
```

Ответ (успех):
```json
{
    "success": true,
    "message": "Регистрация успешна"
}
```

### GET `/api/check_auth.php?token=...`
**Проверка валидности токена**

Ответ (успех):
```json
{
    "authenticated": true,
    "username": "demo",
    "userId": 1
}
```

### POST `/api/logout.php`
**Выход из системы**

Запрос:
```json
{
    "token": "abc123..."
}
```

Ответ:
```json
{
    "success": true,
    "message": "Вы вышли из системы"
}
```

### GET `/api/save_albums.php?year=...&title=...`
**Добавление нового альбома**

Ответ:
```json
{
    "success": true
}
```

### POST `/api/save_albums.php`
**Сохранение всего списка альбомов (для удаления)**

Запрос:
```json
{
    "albums": [
        {"year": "2026", "title": "чао"},
        {"year": "2025", "title": "пока лето не кончилось"}
    ]
}
```

## Как использовать

### 1. Запуск сайта
- Откройте `http://localhost/lab4/` в браузере
- Будет выполнен редирект на страницу входа

### 2. Вход с демо-аккаунтом
- Имя: `demo`
- Пароль: `demo123`
- Нажмите кнопку "Войти"

### 3. Просмотр альбомов
- На главной странице отображена таблица альбомов
- Данные загружаются из `data/albums.json`

### 4. Добавление альбома
- Заполните форму "Добавить альбом"
- Укажите год (4 цифры) и название
- Нажмите "Добавить"
- Альбом появится в таблице

### 5. Удаление альбома
- Найдите альбом в таблице
- Нажмите кнопку "Удалить"
- Подтвердите удаление

### 6. Регистрация нового пользователя
- На странице входа нажмите вкладку "Регистрация"
- Заполните все поля
- Нажмите "Зарегистрироваться"
- После регистрации вернитесь на вкладку "Вход"
- Войдите с новыми учетными данными

### 7. Выход из системы
- Нажмите кнопку "Выход" в правом верхнем углу
- Будет выполнен редирект на страницу входа

## Технологические особенности

### На стороне клиента (JavaScript)
- Валидация форм без внешних библиотек
- AJAX-запросы с использованием Fetch API
- Управление состоянием приложения через localStorage
- Обработка ошибок и сообщения пользователю

### На стороне сервера (PHP)
- RESTful API endpoints
- Валидация данных
- Работа с JSON файлами
- Управление токенами сессий
- Безопасность (проверка входных данных)

### Стили (CSS)
- Адаптивный дизайн (мобильные устройства)
- Bootstrap-like стилизация таблиц
- Градиентные фоны
- Анимации и переходы
- Темная навигация с голубыми акцентами

## Безопасность

⚠️ **Важно**: Это учебный проект. В production среде необходимо:
- Использовать HTTPS
- Хешировать пароли (bcrypt, argon2)
- Реализовать CSRF защиту
- Использовать HttpOnly cookies вместо localStorage
- Валидировать все данные на сервере
- Использовать подготовленные запросы к БД вместо JSON файлов
